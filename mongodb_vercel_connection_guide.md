# MongoDB Atlas & Vercel Connection Guide

Follow these steps to finish setting up your database and connect it to your live Vercel site.

## Phase 1: MongoDB Atlas Setup

### 1. Create your Free Cluster
1.  Sign in at [MongoDB Atlas](https://www.mongodb.com/cloud/atlas).
2.  Click **Create a Deployment** and select the **M0 FREE** tier.
3.  Choose your cloud provider (e.g., AWS) and region, then click **Create**.

### 2. Create a Database User
1.  In the Atlas dashboard, go to **Security** -> **Database Access**.
2.  Click **Add New Database User**.
3.  Choose **Password** as the authentication method.
4.  **Username:** `admin` (or anything you like).
5.  **Password:** Click **Autogenerate Secure Password** and **COPY IT** immediately.
6.  Ensure "Database User Privileges" is set to "Read and write to any database".
7.  Click **Add User**.

### 3. Whitelist IP Addresses
1.  Go to **Security** -> **Network Access**.
2.  Click **Add IP Address**.
3.  Click **Allow Access From Anywhere** (it will enter `0.0.0.0/0`).
    *   *Note: This is required for Vercel because its servers change IP addresses constantly.*
4.  Click **Confirm**.

### 4. Get your Connection String
1.  Go to **Deployment** -> **Database**.
2.  Click the **Connect** button on your cluster.
3.  Choose **Drivers**.
4.  Copy the long connection string. It looks like:
    `mongodb+srv://admin:<password>@cluster0.abcde.mongodb.net/?retryWrites=true&w=majority`

---

## Phase 2: Connecting to Vercel

### 1. Prepare the Final String
Take the string you copied above and replace `<password>` with the password you autogenerated in Phase 1, Step 2.

### 2. Add to Vercel Settings
1.  Open your project in the [Vercel Dashboard](https://vercel.com/dashboard).
2.  Navigate to the **Settings** tab at the top.
3.  Select **Environment Variables** in the left sidebar.
4.  Add a new variable:
    *   **Key:** `MONGO_URI`
    *   **Value:** (Your final connection string with the password)
5.  Add another variable:
    *   **Key:** `JWT_SECRET`
    *   **Value:** (Any random strong string, e.g., `SuperSecretERP123!`)
6.  Click **Save**.

### 3. Apply the Changes
1.  Go to the **Deployments** tab in Vercel.
2.  Find your latest deployment, click the three dots `...` on the right.
3.  Select **Redeploy**.
4.  Check "Redeploy with existing build" if it asks (or just let it build fresh).

---

## Phase 3: Final Local Setup (Optional)
To run it on your own computer, open your `backend/.env` file and paste the same `MONGO_URI` (with the password) there.
